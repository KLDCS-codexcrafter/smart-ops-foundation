## 4. Card-level verdicts

Format per card: **Operix ships** (verified file path) → **closest peer per vendor** (named module) → **honest differentiator OR gap** (qualitative, no scores).

> Where the *verdict* is the same as the row in §3 above, only the differentiator/gap line is repeated here for brevity. The named peer modules in §3 are not re-listed.

1. **Command Center.** Operix ships a cross-card SSOT replica model in `command-center-shell-config.ts` whose masters are referenced by every other card. **Differentiator:** TallyPrime/Marg/Busy have no equivalent MDG concept; SAP MDG and Oracle PDM/PIM are heavier and licensed separately. **Gap:** no Operix data-quality scoring / steward workflows that MDG and Oracle ship out-of-the-box.

2. **Procure360.** `Procure360Page.tsx`. **Differentiator:** integrated 3-mode vendor selection inside the Indian compliance shell (MSME 43BH already wired via Vendor Portal). **Gap:** no contract-lifecycle (CLM) sub-module like SAP Ariba CLM or Oracle Procurement Contracts.

3. **Main Store Hub.** `MainStoreHubPage.tsx`. **Gap:** route declared as `/erp/main-store-hub` in the registry while the folder is still `src/pages/erp/inventory/`. Functionality on par with SAP MM-IM / NetSuite Inventory for SME volumes; no native EWM-class slotting/wave-picking.

4. **QualiCheck.** Folder present; entry via shell-config. **Differentiator:** vs Tally/Marg/Busy/Zoho — QualiCheck is a real QMS with NCR/CAPA/MTC/FAI, none of those four ship one. **Gap:** no formal ISO/IEC document control linkage (DocVault adjacency exists, integration not exposed in card).

5. **GateFlow.** `GateFlowPage.tsx`. **Differentiator:** native weighbridge integration is rare — SAP Yard Logistics, Oracle Transportation, and D365 Warehouse do not include it; usually a third-party. **Gap:** ANPR/camera integrations not benchmarked.

6. **Production.** `ProductionPage.tsx`. **Gap:** no MRP-II grade finite scheduling like SAP PP-DS or Oracle ASCP/Manufacturing Cloud; OEE captured but no MES connector library.

7. **MaintainPro.** `MaintainProPage.tsx`. **Differentiator:** EAM/CMMS scope absent in Tally/Marg/Busy/Zoho/NetSuite (NetSuite needs a partner CMMS). **Gap:** no IoT condition-monitoring ingestion comparable to SAP Asset Performance Mgmt or Oracle IoT.

8. **RequestX.** `RequestXPage.tsx`. **Differentiator:** 11 indent categories + 3-tier approval matrix is broader than Tally/Marg/Busy single-form indents and matches what SAP SRM/Ariba and Oracle SSP offer in concept. **Gap:** no catalog-punchout to external supplier sites.

9. **EngineeringX.** `EngineeringXPage.tsx`. **Differentiator:** AI similarity / change-impact prediction is an Operix-specific feature; SAP EPD and Oracle PLM ship the PLM data model but not the similarity scoring. **Gap:** no CAD viewer plug-ins (DWG/STEP) listed in the card.

10. **Department Stores.** `DepartmentStorePage.tsx`. Functionally equivalent to multi-warehouse capabilities in every vendor; **Differentiator:** UI is a lightweight operator console (not the inventory backbone) — a separation NetSuite/D365/Odoo do not make.

11. **Vendor Portal.** `VendorPortalPage.tsx`. **Differentiator:** Saathi (साथी) WhatsApp AI co-pilot for vendors — not offered by any of the 8 competitors at the time of writing. **Gap:** no network/Ariba-equivalent multi-tenant supplier discovery.

12. **SiteX.** `SiteXPage.tsx`. **Differentiator:** unified site console combining DPR + snag list + PTW + JSA + Toolbox Talks + site imprest — SAP and Oracle split these across PS + EHS + Treasury; NetSuite and BC do not natively have EHS. **Gap:** no PWA/offline mode listed for site-app reliability on poor connectivity.

13. **Logistics.** `UNVERIFIED` standalone page — no `src/pages/erp/logistics/` folder on HEAD; registry comment claims live route since `T-Phase-1.1.1p-v2`. **Gap (process):** card should resolve to a verifiable file before any further claim is made.

14. **SalesX Hub.** Folder present; entry via shell-config. **Differentiator:** 6-role mobile suite (Salesman, Telecaller, Supervisor, Manager, Distributor, Customer) is broader than the 1-2 mobile personas Tally/Marg/Busy/Zoho ship. **Gap:** no quoting CPQ engine of the depth SAP CPQ or Oracle CPQ Cloud offer for configurable products.

15. **Distributor Hub.** `DistributorHubPage.tsx`. **Differentiator:** scheme management + secondary sales tracking — Marg's distributor edition is the only direct Indian peer; SAP/Oracle/NetSuite/D365 treat distributors via generic channel-management add-ons.

## 5. Cross-suite advantages (qualitative)

A1. **India-statutory depth packaged inside the ERP shell, not as add-ons.** GST/TDS/E-invoice/E-way bill (Fin Core), MSME 43BH stop-clock (PayOut + Vendor Portal), FEMA 270-day timer (EximX), MCA Rule 3 audit trail (Fin Core). Global vendors (SAP/Oracle/NetSuite/D365) deliver Indian localisation as separate licensed packs; Tally/Zoho/Marg/Busy cover GST well but lack the breadth (no FEMA, no MSME-43BH stop-clock workflow, no PT/LWF inside payroll for Tally/Marg/Busy).

A2. **Vendor & distributor portals with WhatsApp AI co-pilots** (Saathi). No competitor in the benchmarked list ships a WhatsApp-native vendor co-pilot.

A3. **Project orchestrator stamping (`project_centre_id`) across every transaction card.** Real cross-card project P&L without re-keying — closer in concept to SAP PS than to NetSuite SuiteProjects or Odoo Projects.

A4. **Site-execution unified console (SiteX)** combining DPR + snag list + PTW + JSA + site imprest. SAP/Oracle split these across PS+EHS+Treasury; mid-market vendors do not have EHS at all.

A5. **EcomX claims engine** for Indian marketplace settlements (194-O TDS + GST-TCS). Competitors offer connectors, not the claims-recovery workflow.

A6. **Read-only architectural lock on the analytics layer (InsightX)** enforced by code comments — a governance discipline not built into competing BI add-ons.
